# Микросервисная архитектура распределенного приложения

## Описание проекта

Этот проект представляет собой распределенное приложение, состоящее из двух микросервисов, реализующих взаимодействие через REST API. Один из сервисов отвечает за аутентификацию и выдачу JWT токенов, а второй сервис — за работу с данными, доступ к которым осуществляется только для авторизованных пользователей.

Проект демонстрирует принципы микросервисной архитектуры, где каждый сервис имеет свою собственную базу данных, а также реализует базовые механизмы безопасности, такие как аутентификация с использованием токенов и шифрование данных при передаче.

## Архитектура

Проект состоит из следующих частей:

1. **Auth Service (Сервис аутентификации)**  
   Этот сервис генерирует JWT токены для пользователей. Он отвечает за проверку учетных данных пользователя и выдает токен для дальнейшего использования в других микросервисах.
   
2. **Data Service (Сервис данных)**  
   Этот сервис предоставляет защищенные данные только для авторизованных пользователей. Он использует JWT токен для аутентификации и проверяет его подлинность, делая запросы к сервису аутентификации.

3. **PostgreSQL база данных**  
   Каждый микросервис использует свою собственную базу данных. В частности, **Data сервис** использует PostgreSQL для хранения данных.

## Установка

### 1. Установка зависимостей

Для работы с проектом вам нужно установить зависимости. Если у вас уже установлен Go, выполните следующие команды:

```bash
go get -u github.com/gin-gonic/gin
go get -u github.com/jinzhu/gorm
go get -u github.com/jinzhu/gorm/dialects/postgres
go get -u github.com/dgrijalva/jwt-go
```
